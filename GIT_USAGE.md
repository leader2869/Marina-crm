# Руководство по использованию Git для отката изменений

## Текущее состояние

Все изменения сохранены в Git репозиторий. Создан тег `v1.1.0` для текущей версии.

## Основные команды для отката

### 1. Просмотр истории коммитов
```bash
git log --oneline
```

### 2. Просмотр всех тегов
```bash
git tag -l
```

### 3. Откат к предыдущему коммиту (сохраняет изменения)
```bash
git reset --soft HEAD~1
```

### 4. Откат к предыдущему коммиту (удаляет изменения)
```bash
git reset --hard HEAD~1
```

### 5. Откат к конкретному коммиту
```bash
git reset --hard <commit-hash>
```

### 6. Откат к тегу (например, v1.0.0)
```bash
git checkout v1.0.0
```

### 7. Создание новой ветки от тега (рекомендуется)
```bash
git checkout -b restore-v1.0.0 v1.0.0
```

### 8. Возврат к последнему коммиту
```bash
git checkout master
# или
git checkout main
```

## Примеры использования

### Откат к версии v1.0.0
```bash
# Создать новую ветку от тега
git checkout -b restore-v1.0.0 v1.0.0

# Или просто переключиться на тег
git checkout v1.0.0
```

### Откат к последнему коммиту
```bash
git reset --hard HEAD
```

### Просмотр изменений перед откатом
```bash
git diff HEAD~1
```

### Откат конкретного файла
```bash
git checkout HEAD -- <путь-к-файлу>
```

## Важные замечания

⚠️ **ВНИМАНИЕ**: Команды `git reset --hard` и `git checkout` могут удалить незакоммиченные изменения!

⚠️ Перед откатом рекомендуется:
1. Сохранить текущие изменения: `git stash`
2. Или создать новую ветку: `git checkout -b backup-branch`

## Текущие теги

- `v1.0.0` - Начальная версия с управлением пользователями
- `v1.1.0` - Версия с поиском и редактированием судов и пользователей

## Создание резервной копии перед откатом

```bash
# Сохранить текущие изменения
git stash save "Backup before rollback"

# Выполнить откат
git reset --hard <commit-hash>

# Вернуть изменения (если нужно)
git stash pop
```
